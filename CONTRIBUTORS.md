## How to Contribute

We actively welcome contributors at all levels:

- **Code contributors** — PR bug fixes, features, improvements
- **Documentation** — README, guides, tutorials, examples
- **Testing** — Find bugs, suggest edge cases, write tests
- **Ideas** — Feature requests, integration suggestions, real-world feedback
- **Advocacy** — Star the repo, share with your network, use it in production

**First time contributing?** Check [COMMUNITY.md](./COMMUNITY.md) and [docs/CONTRIBUTING.md](./docs/CONTRIBUTING.md).

## How to Add Yourself

After your first contribution (PR merged, issue resolved, documentation improved):

1. **Submit a PR** adding yourself to this file
2. **Include:**
   - Your GitHub username
   - Contribution area (code/docs/testing/ideas)
   - Brief description (1-2 sentences)

Example:
```markdown
- **Your Name** (@your-github) — Added Twitter discovery source, improved hook scoring docs
```

Or email Steve and we'll add you.
